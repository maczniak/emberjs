Planning The Application Edit Page

크기는 작지만 TodoMVC는 현대적인 실글 페이지 어플리케이션에 전형적인 기능 대부분을 포함한다. 먼저 잠시 사용자 입장에서 TodoMVC가 어떻게 동작하는지 살펴보자.

TodoMVC의 주요 기능은 다음과 같다.

  1. 할일 목록을 사용자에게 보여준다. 사용자가 할일을 추가하거나 삭제하면 이 목록이 늘어나거나 줄어든다.

  2. <input>으로 새로운 할일 항목을 입력받는다. <enter> 키를 누르면 항목을 새로 만들어서 아래 목록에 보여준다.

  3. 할일 각각의 완료 상태를 변경하는 체크박스이다. 할일을 새로 만들면 미완료 상태로 시작한다.

  4. 미완료 할일 개수를 보여준다. 할일이 새로 추가하거나 기존의 할일을 완료하면 이 개수를 갱신한다.

  5. 모든 할일, 미완료 할일, 완료된 할일 사이를 오갈 때 사용하는 링크이다.

  6. 완료된 할일을 모두 삭제하는 버튼이며 사용자에게 완료된 할일 개수를 알려준다. 완료된 할일이 없다면 이 버튼이 안보인다.

  7. 특정 할일 하나를 삭제하는 버튼이다. 사용자가 할일 위로 마우스를 올리면, 이렇게 붉은 X 형태로 버튼이 보인다.

  8. 기존의 할일의 완료 상태를 모두 변경하가는 체크박스이다. 그리고 모든 할일이 완료 상태라면, 자동으로 이 체크박스가 선택된다.

  9. 사용자가 더블클릭하면 개별 할일을 편집할 수 있는 텍스트필드를 보여준다. <enter> 키를 누르거나 포커스를 텍스트필드 밖으로 옮기면 변경한 글을 반영한다.

  10. 브라우저의 localstorage 기능을 사용하여 어플리케이션을 다시 읽어들여도 사용자의 할일을 유지한다.

TodoMVC 사이트에서 완성된 어플리케이션을 사용해볼 수 있다.
